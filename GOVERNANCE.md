# Project Governance

This document outlines the governance model and decision-making process for the **Ultimate Torrent Tracker Aggregator Pro Max** project.

## Roles and Responsibilities

### 👑 Lead Architect (Creator / Maintainer)
- **Mohammad Munna ([@mrgusux](https://github.com/mrgusux))**
- Holds the final decision-making authority on all architectural changes, feature integrations, code reviews, and releases.
- Manages repository access, security policies, and community guidelines.

### 🛠️ Contributors
- Anyone who submits a merged Pull Request, reports a valid issue, or contributes to project documentation.
- Contributors are highly encouraged to participate in discussions, propose new tracker sources, and improve the 12-stage sanitization pipeline.

## Decision Making Process

Currently, this project operates under the **"Benevolent Dictator For Life" (BDFL)** model. This means that while community feedback, ideas, and pull requests are highly valued and actively sought, all final technical and directional decisions rest with the Lead Architect.

For major changes (e.g., modifying the core ShellCheck rules or changing the automated GitHub Actions schedule), discussions will be held openly via GitHub Issues to gather community feedback before final implementation.

## Code of Conduct
All participants in the governance and decision-making process are required to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Amendments
Changes to this governance model can be proposed by anyone via Pull Requests but must be explicitly approved and merged by the Lead Architect.
