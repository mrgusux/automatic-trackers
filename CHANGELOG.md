## [2.0.0] - 2026-06-12

Complete architecture overhaul ("v2.0").

### Added
- **Blacklist exclusion filtering:** known-bad trackers are fetched from dedicated `BLACKLIST_SOURCES` (including ngosang's official blacklist), published to an accumulative `blacklist.txt`, and removed from every final list by exact URL and host:port matching.
- True parallel fetching (`xargs -P`) for dramatically faster runs.
- Persistent fetch cache between CI runs via `actions/cache`, used strictly as a fallback when a source is unreachable (fresh data always preferred).
- Safety gates: minimum tracker count (`MIN_TRACKER_COUNT`) and a 60% drop guard — a degraded run can never overwrite good lists.
- Corrupted-data recovery in the sanitizer: comma-joined lines are split, double-scheme/glued URLs dropped, stray quotes stripped, double slashes collapsed.
- Five new high-quality tracker sources (ngosang GitHub Pages mirrors, newtrackon 90% API, jsdelivr best-list mirrors).
- Real configuration validation: duplicate or malformed source URLs fail the run before fetching.
- bats regression tests for every sanitizer rule and blacklist filter, mirroring the exact pipeline logic.
- `actionlint` in CI and pre-commit — workflow YAML and inline bash are now linted.
- PR size labels, `security`/`community` path labels, and stale-bot exemptions for security-labeled items.
- Release assets now include checksums (`RELEASE_SHA256SUMS.txt`); all assets are Sigstore-signed with signature, certificate, and bundle.
- Discord failure alerts now include run link, branch, commit, and actor via a dependency-free webhook embed.

### Changed
- JSON API files are generated safely with `jq` instead of string concatenation; `stats.json` gains a `blacklist_removed` field.
- All workflows hardened: top-level read-only permissions, job-level grants, concurrency groups, and timeouts.
- `trackers.txt` is now maintained as a mirror of `all_trackers.txt` for backward compatibility with existing hotlinks.
- `SHA256SUMS.txt` is regenerated by the main engine itself (bot pushes cannot trigger the checksum workflow).
- Local pre-commit checks now match CI exactly (ShellCheck severity, actionlint).
- Issue/PR/Discussion templates rebuilt with duplicate-prevention checklists, category dropdowns, and accurate file references.
- All community documents cross-linked and synced with the real repository structure.

### Fixed
- Discord failure alert never triggered (wrong workflow name and a stray character in the YAML).
- `SHA256SUMS.txt` included its own hash, causing endless self-update commits.
- SBOM workflow used a command incapable of generating SBOMs; replaced with Syft (CycloneDX).
- Artifact signing produced signatures without certificates (unverifiable) and signed checkout files instead of the actual published release assets.
- Silent failure masking (`|| true`) removed from commit/push steps; pushes now retry with rebase and fail loudly.
- Two blacklist URLs were wrongly used as tracker sources, injecting known-bad trackers into the final lists.
- In-workflow cache lived in `/tmp` and could never persist between runs.
- `MIN_TRACKER_COUNT` and `MAX_PARALLEL_JOBS` were defined but never used.
- Test suite referenced non-existent files (`scripts/update.sh`, `config/sources.txt`) and always failed.
- PR template and labeler rules pointed at a non-existent `config/` directory.
- Tracker output corruption: comma-joined mega-lines, double-scheme URLs, and glued URLs are now filtered out.

### Removed
- Blacklist URLs from the valid source list (relocated to `BLACKLIST_SOURCES` — no URL was deleted).
- Dead in-workflow backup logic that restored a snapshot nothing ever read.
- Unused `pull-requests: write` permission and unpinned third-party `@master` actions.

## [1.0.0] - 2024-01-01

### Added
- Initial release: automated tracker aggregation from community sources every 6 hours via GitHub Actions.
- Sanitization pipeline: BOM/control-character stripping, protocol validation, RFC-1918 private IP filtering, deduplication.
- Rotating User-Agent fetching with exponential backoff retries.
- SHA256 idempotency check to prevent redundant commits.
- Plain-text outputs (`all_trackers.txt`, `udp.txt`, `http.txt`, `https.txt`, `ws.txt`) and JSON API files.
