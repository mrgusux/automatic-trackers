## [2.0.0] - 2026-06-11

Complete architecture overhaul ("v2.0").

### Added
- **Blacklist exclusion filtering:** known-bad trackers are fetched from dedicated blacklist sources, published to `blacklist.txt`, and removed from every final list.
- `scripts/update.sh` — single source of truth for all aggregation logic, shared by CI, Docker, and local runs.
- Config-driven source lists: `config/sources.txt` and `config/blacklist_sources.txt`, with automatic CI validation (no duplicates, valid URLs only).
- True parallel fetching (`xargs -P`) for dramatically faster runs.
- Persistent fetch cache between CI runs via `actions/cache`.
- Minimum tracker count safety gate (`MIN_TRACKER_COUNT`) — a degraded run can never overwrite good lists.
- New CDN mirror and live API sources for redundancy.
- bats test suite wired into CI, including blacklist filtering tests.

### Changed
- JSON API files are now generated safely with `jq` instead of string concatenation.
- Docker image rebuilt: non-root user, lean package set, OCI labels, correct entrypoint.
- Docker Compose: working resource limits, security hardening, one-shot semantics, persistent cache volume.
- Makefile: self-documenting help, safe `clean`, real `lint` and `test` targets.
- All community documents (SUPPORT, SECURITY, GOVERNANCE, MAINTAINERS, SOURCES, CONTRIBUTING) rewritten with accurate, cross-linked content.

### Fixed
- Docker entrypoint pointed to a non-existent script (`workflows/update.sh`).
- Discord failure alert never triggered (wrong workflow name and a stray character in the YAML).
- `SHA256SUMS.txt` included its own hash, causing endless self-updates.
- SBOM workflow used a command incapable of generating SBOMs; replaced with Syft.
- ShellCheck lint job had no shell files to scan; it now lints the real pipeline script.
- `.gitignore` ignored files that the workflow is required to commit.
- Silent failure masking (`|| true`) removed from commit/push steps.

### Removed
- Orphaned `trackers.txt` (never generated by any workflow).
- Dangerous `make push` target and destructive `make clean` wildcard.
- Dead in-workflow cache and backup logic that could never persist between runs.

## [1.0.0] - 2024-01-01

### Added
- Initial release: automated tracker aggregation from community sources every 6 hours via GitHub Actions.
- Sanitization pipeline: BOM/control-character stripping, protocol validation, RFC-1918 private IP filtering, deduplication.
- Rotating User-Agent fetching with exponential backoff retries.
- SHA256 idempotency check to prevent redundant commits.
- Plain-text outputs (`all_trackers.txt`, `udp.txt`, `http.txt`, `https.txt`, `ws.txt`) and JSON API files.
